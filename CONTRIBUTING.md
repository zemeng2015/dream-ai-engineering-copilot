# Contributing to DREAM

Thank you for considering a contribution. DREAM is an Apache-2.0 open-source
project intended to stay cleanly reusable across personal, open-source, and
company-import settings.

## License of Contributions

Unless you explicitly state otherwise in writing, any contribution intentionally
submitted to this repository is submitted under the Apache License, Version 2.0.
This matches Section 5 of the Apache-2.0 license.

Use a Developer Certificate of Origin sign-off on commits:

```text
Signed-off-by: Your Name <your.email@example.com>
```

You can add it with:

```bash
git commit -s
```

## Clean Contribution Rules

- Do not contribute employer-owned code unless you have written authorization.
- Do not contribute confidential business data, internal tickets, runbooks,
  logs, cloud paths, repository names, API endpoints, credentials, or customer
  information.
- Do not copy code from private repositories or proprietary systems into this
  project.
- Do not add real company knowledge packs, connectors, prompts, or deployment
  configuration to the public repository.
- Synthetic examples, public-domain facts, original code, and properly licensed
  open-source-compatible contributions are acceptable.

## Company Forks and Private Extensions

Companies may import DREAM under Apache-2.0 and develop private forks or private
extensions. Keep company-specific changes, knowledge packs, connectors, prompts,
and deployment configuration in company-controlled repositories unless the
company explicitly approves upstream contribution and the contribution contains
no confidential material.

## Pull Request Expectations

- Keep changes focused and reviewable.
- Add or update tests for behavior changes.
- Keep generated artifacts, local databases, credentials, and build outputs out
  of commits.
- Preserve existing license and attribution notices.
